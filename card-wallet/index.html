Card Wallet

Card Wallet is a macOS app built with SwiftUI for managing, visualizing, and organizing your credit cards.

Xcode 15 or later
View Project on GitHub

Explore Your Card Collection

Browse your card library in a grid that reflows as the window resizes, with search and sort built in.

Grid View

The card library uses a LazyVGrid that reflows automatically as the window resizes, keeping spacing consistent at any size.

| |

Search & Sort

A .searchable modifier filters cards by name. Sorting is handled through a Picker in the toolbar menu: open date, credit limit, or name.

| |

Credit Card Categories

The sidebar uses an enum to categorize cards as Personal, Business, Closed, or All, with filter logic that checks card properties directly.

| |

Dive Into Card Details

Each card has a detail view showing its properties and history, with SwiftUI navigation and data binding keeping everything in sync.

Detail View

Double-clicking a card opens a DetailView showing its name, type, credit limit, and associated bank. Properties are grouped with GroupBox and VStack. The view scrolls and handles any amount of content.

| |

Show History

The history section shows a chronological list of changes for each card, including modifications, promotions, and status updates, stored in a related SwiftData model.

| |

Add and Manage Cards

Add, edit, and delete cards through modals built with SwiftUI’s Form component, backed by SwiftData for persistent storage.

Adding, Editing, Deleting

Card management is handled via modals that open when adding or editing a card. The data is stored using SwiftData and updates are reflected in real time. Deletions prompt a confirmation alert, using SwiftUI’s native .alert and .sheet behaviors.

| |

Form Breakdown

The form includes fields for basic card details like name, credit limit, and bank. Toggles are used for business and charge card status. The layout is built with SwiftUI’s Form, using @Binding to connect user input directly to the model.

Visualize Your Data

Bar and pie charts break down credit limits, card age, and bank distribution, built with the Swift Charts framework.

Charts View

The Charts view uses bar and pie charts to show limit distribution, card age, and bank breakdown. Charts are arranged in a grid and pull directly from SwiftData queries.

| |

Validate With UI Tests

An automated XCUITest suite covers the main user flows end to end.

Component-Driven Automation

Each part of the app has its own screen object (Grid, Toolbar, Detail, Form) matching how the app is structured. These compose into full user flows while keeping tests readable and reusable.

Accessibility-First Selectors

Selectors map directly to accessibility identifiers set in SwiftUI views. Each interaction wrapper checks that a control exists and is enabled before tapping, catching regressions like missing buttons or disabled actions early.
